# 第8章：自動微分 (Automatic Differentiation)

ニューラルネットワークの学習や物理シミュレーションのパラメータ同定において、関数の**勾配（Gradient）**を計算することは不可欠です。Flunoは、強力な**自動微分 (Automatic Differentiation, AD)** エンジンを内蔵しており、これを「微分可能プログラミング (Differentiable Programming)」として提供します。

## 8.1 ADFloat 型

Flunoにおける浮動小数点数は、単なる `f64` ではなく `ADFloat` という型として実装されています（推論モード時）。この型は、数値だけでなく、その数値がどのような計算を経て生成されたかという**計算グラフ（Computational Graph）**の情報を保持できます。

```rust
// 通常の計算のように見えるが、内部でグラフが構築されている
fn energy(x: ADFloat) -> ADFloat {
    return x * x + 3.0 * x + 1.0;
}
```

## 8.2 勾配の計算

ある関数 `f(x)` の `x` における微分係数 `f'(x)` を求めたい場合、Flunoでは以下のように記述します（概念コード）。

```rust
let x = 2.0; // 具体的な入力値

// 勾配計算モードで実行
let (y, grad) = value_and_grad(energy, x);

println("Value: {}", y);  // 2^2 + 3*2 + 1 = 11
println("Grad:  {}", grad); // 2*x + 3 = 2*2 + 3 = 7
```

Flunoは **Reverse-mode AD (Backpropagation)** を採用しているため、多数の入力パラメータを持つ関数に対しても効率的に勾配を計算できます。これはディープラーニングの学習において必須の特性です。

## 8.3 確率推論との統合

Flunoにおいて、ADは確率的プログラミング (PPL) を支える基盤技術でもあります。

*   **HMC (Hamiltonian Monte Carlo)**: 確率密度関数の勾配を使って、効率的にサンプリングを行います。
*   **VI (Variational Inference)**: 変分パラメータ（ELBO）の勾配を使って、確率分布の形を最適化します。

ユーザーは通常、ADを直接意識する必要はありません。`infer_hmc` などの推論関数を呼び出すだけで、Flunoが裏側で自動的に計算グラフを構築し、高速な勾配計算を実行してくれます。

## 8.4 テンソルとGPUサポート

（将来の展望）
現在、Flunoはスカラー値に対するADをサポートしていますが、将来的には多次元配列（Tensor）に対するADと、GPUバックエンドによる高速化を計画しています。これにより、PyTorchのような深層学習フレームワークと同等の機能を、より安全で統合された言語環境で利用できるようになります。
